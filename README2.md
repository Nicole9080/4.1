## Вариант №15 — Страховая компания

*В ходе выполнения работы были разработаны два дочерних класса для Policy:*

**AutoInsurance** - автострахование (Атрибуты: car_model - Модель автомобиля, car_year - Год выпуска)

**HealthInsurance** - медицинское страхование (Атрибуты: coverage_type - Тип покрытия, hospital_list - Список клиник)

*И один дочерний класс для Policyholder:*

**FamilyPlanHolder** - клиент с семейным планом (Атрибуты: family_members - Количество членов семьи, is_vip - VIP-статус)

В конструкторах дочерних классов были использованы super().__init__(...) для инициализации общих полей (policy_id, holder_name, premium_amount для дочерних классов БК1 и client_id, full_name, phone для дочернего класса БК2) и инициализированы специфичные поля.

В качестве полиморфного метода был реализован метод **calculate_payout(damage)**, который рассчитывает страховую выплату по ущербу для разных типов страхования с разными процентами покрытия:

Для AutoInsurance выплата составляет 80% от ущерба

Для HealthInsurance выплата составляет 90% от ущерба

Был реализован класс-контейнер InsuranceCase, конструктор которого принимает объект Policyholder (клиента) и создаёт пустой список для объектов Policy (полисов). Также был реализован метод add_item(item), который добавляет объекты в список. В классе-контейнере был реализован метод get_total_payout(damage), который перебирает список добавленных объектов, вызывает у каждого объекта его полиморфный метод и суммирует выплаты. Если клиент является FamilyPlanHolder, применяется семейная скидка (10% для семьи из 2-3 человек, 15% для семьи из 4 и более человек), а для VIP-клиентов дополнительно предоставляется скидка 5%.

В демонстрационной части программы созданы экземпляры дочерних классов: объекты автострахования и медицинского страхования; клиенты (обычный, с семейным планом, VIP с семейным планом); созданы страховые случаи, в которые были добавлены полисы, и выполнен вывод оформленной в отчёт информации о них в консоль.
